1 Confusing Tap Gestures

Current Behavior Double-tap to start exercise, single-tap to edit details
Problem Non-standard interaction pattern. No iOS app uses double-tap as primary action. Users don't discover how to start exercises.
Recommended Fix
  • Single-tap → Start exercise timer immediately
  • Long-press → Show context menu (Edit, Delete, Reorder)
  • Alternative: Make the play button more prominent and remove tap gestures from card body

2 Missing Onboarding

Current Behavior User drops directly into workout list on first launch
Problem HIT training has specific methodology (eccentric/concentric phases). Users need guidance to understand the app's purpose and how to use it effectively.
Recommended Fix Add first-launch onboarding flow:
  • Welcome screen explaining HIT training benefits
  • Experience level picker (reuse existing UI from Profile)
  • Quick tutorial showing phase-based timer concept
  • Optional: "Let's calibrate your starting weights" wizard

3 Weight Entry Flow

Current Behavior User logs weight AFTER completing the exercise set
Problem Users must remember the weight they used. Cognitive load during/after intense exercise. No weight progression suggestions.
Recommended Fix
  • Pre-populate weight input BEFORE starting timer
  • Show "Last time: 135 lbs" prominently on exercise card
  • Quick +5/-5 adjustment buttons before starting
  • Auto-suggest weight progression: "You hit failure at 135 lbs last time. Try 140 lbs?"
  • After completion, just confirm or adjust the pre-set weight

4 Timer View Overwhelm

Current Behavior Timer view shows: phase indicators (E-C-F), large countdown, phase name, play/pause/skip controls, all simultaneously
Problem Information overload during physically demanding exercise. Users can't focus on form while processing UI elements.
Recommended Fix Simplify the timer view:
  • Single large timer number (make it HUGE)
  • Phase name in large text below
  • Remove or minimize the E-C-F phase indicators
  • Add pulsing/breathing animation to current phase
  • Strong haptic feedback at phase transitions
  • Consider: Full-screen color change per phase (blue for eccentric, green for concentric)

6 Exercise Reordering

Current Behavior Exercises are in fixed order from template/defaults
Problem Users need to adapt workout order based on equipment availability at the gym
Recommended Fix
  • Add drag handles on exercise cards
  • Or: "Edit Order" button that enables drag-to-reorder mode
  • Save custom order per workout type

7 Progress Feedback & Celebrations

Current Behavior Checkmark when exercise complete, PR badge shown in history view
Problem Instant gratification is missing. Users don't feel rewarded for achievements in the moment.
Recommended Fix
  • Confetti animation when hitting a new PR
  • "ğŸ”¥ 3 sets completed!" toast notifications
  • Progress ring on workout list header showing % complete
  • Streak celebration animation at 7, 14, 30 days
  • Sound effect options for PR achievement
  • "Personal Best!" callout immediately after logging weight

8 Simplified Stats Tab

Current Behavior 4 sections: Overview, Charts, Tools, Body - horizontal scrolling picker
Problem Too many options leads to decision fatigue. Most users just want to see "am I getting stronger?"
Recommended Fix
  • Default to Overview with key metrics only
  • Surface the most motivating stat prominently: "You're 12% stronger than last month"
  • Collapse Charts, Tools, Body into expandable sections or "More Stats" button
  • Show the single most relevant PR at the top

9 Haptic Feedback

Add iOS haptics throughout the app:

  • Phase transitions - Heavy impact tap
  • 10-second warning - Light double-tap
  • Set completion - Success haptic pattern
  • PR achievement - Celebration haptic pattern
  • Button presses - Light tap

10 Watch & Lock Screen Widgets

Extend presence beyond the app:

  • Apple Watch complication showing next workout date
  • Lock Screen widget with current streak
  • Live Activity during workout showing current phase/time
  • "Time to train!" notification with action buttons

11 Quick Actions (3D Touch / Long Press on App Icon)

  • "Start Workout A"
  • "Start Workout B"
  • "Continue Last Workout"
  • "Log Body Weight"

12 Accessibility Improvements

  • VoiceOver labels for all interactive elements
  • Dynamic Type support (scalable text)
  • Reduce Motion option (disable animations)
  • High contrast mode support
  • Voice control compatibility

13 Appearance Settings

Current App forces dark mode always
Recommended
  • Follow system setting by default
  • Manual override in settings (Light / Dark / System)
  • Keep existing theme color options
